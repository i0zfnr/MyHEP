Outstanding Priority Work
-------------------------
1. Production/debug configuration
   - The local .env uses APP_ENV=local and APP_DEBUG=true.
   - This is acceptable only for local development. Do not expose detailed errors on
     a real deployment or long-lived public Cloudflare tunnel.
   - Production must use APP_ENV=production and APP_DEBUG=false.
   - Keep environment-specific values out of committed application code.

2. Dependency security updates
   - composer audit reported 16 known advisories affecting Laravel, Symfony, Guzzle,
     and related packages, including high-severity advisories.
   - Update dependencies in a dedicated change, then run the full test/build/manual
     verification suite before deployment.

3. QR token race condition
   - File: app/Http/Controllers/Student/MovementController.php
   - Token lookup and token rotation are separate operations. Concurrent scans could
     both validate the same token before it is rotated.
   - Make validation and rotation atomic using a database transaction with
     lockForUpdate(), or use a conditional atomic update.

4. Duplicate active movement race condition
   - File: app/Http/Controllers/Student/MovementController.php
   - The active-movement check and checkout insert are not in one transaction.
   - Concurrent submissions could create multiple active checkout records.
   - Add transaction/locking protection and, if practical, a database-level invariant.

5. Real feature test coverage
   - Existing tests only cover the homepage and a guest redirect.
   - Add tests for login throttling, admin scopes, stale/deleted accounts, password reset
     attempts, QR single use, concurrent movement submissions, CRUD authorization,
     uploads, exports, and EN/MS locale switching.

6. routes/web.php maintainability
   - routes/web.php is approximately 1,858 lines and contains substantial CRUD/business
     logic in closures.
   - Move scholarship, discipline, announcements, fines, stickers, and exports into
     controllers, Form Requests, and focused service classes in small verified stages.

7. AI Helper / agent redesign
   - Current files:
     app/Http/Controllers/Admin/AiHelperController.php
     app/Http/Controllers/Student/AiHelperController.php
     resources/views/admin/ai_helper/index.blade.php
     resources/views/layouts/app.blade.php
     config/services.php
     routes/web.php
   - Current provider priority is Gemini, then OpenAI, then DeepSeek, based on
     configured API keys.
   - Student AI currently redirects to the dashboard with an unavailable message.
   - Keep student AI disabled until privacy boundaries, prompt behavior, safety
     messages, and permissions are defined.
   - Keep admin AI read-only unless a future task adds a reviewed action/approval
     workflow.
   - Do not send secrets, raw session payloads, uploaded files, or full student
     datasets to external AI providers.
   - Add feature tests before enabling student-facing agents or AI-assisted writes.
